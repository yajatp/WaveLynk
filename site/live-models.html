Live Models

Three interactive models of the CCI framework. Adjust channel parameters in real time and observe how the adaptive switching mechanism responds to approaching instability. All computation runs in your browser — no server required.

1

Link Stability Simulator

Configure channel conditions, then run the degradation sequence. Watch throughput and latency respond as CCI approaches the coherence cliff — and observe WaveLynk switch to MRT before the link fails.

Channel Parameters
CSI age τf (ms) 5.0
Coherence time Tc (ms) 20.0
Blockage factor β 0.00
Mobility v (m/s) 1.0
User load K 4
Environment sensitivity α 1.00
Status: Stable — ZF beamforming operating normally.
Live Metrics
CIN = 0.25 Risk = 0.06 SNR = 24.0 dB Mode: ZF
Blue = throughput (Mbps) · Orange = latency (ms) · Red marker = ZF→MRT switch point
What you're seeing: As channel conditions degrade — rising CSI age, increasing mobility, growing blockage — CCI climbs toward the coherence cliff. When it crosses γ = 0.6, WaveLynk switches from ZF to MRT. Throughput drops slightly but stabilizes; latency stops spiking. This is the core value proposition: graceful degradation instead of sudden collapse.
2

CCI Phase Map

Explore how condition number κ, CSI age τ, and Doppler shift fD interact to determine the stability boundary. The color map shows stable (green), transition (yellow), and unstable (red) regions.

Input Parameters
κ (condition number) 5
τ (CSI age, ms) 10
fD (Doppler, Hz) 50
CCI = 0.50 Risk = 0.22 Pcollapse = 0.22
What you're seeing: The 2D map plots τ (x-axis) vs fD (y-axis). Color shows the CCI value for your current κ setting. The dashed line is the CCI = 1 stability boundary. Your operating point is the colored dot — green means stable, red means at risk.
Phase Map — Stability Regions
Green = stable (CCI < 0.5) · Yellow = transition · Red = unstable (CCI ≥ 1) · Dot = current operating point

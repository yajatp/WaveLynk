Hardware Validation

Experimental results — Wi-Fi 7 testbed

Measured using iperf3 on a Netgear Nighthawk Wi-Fi 7 router at 6 GHz, with 2 client laptops and 2 monitor smartphones across 100 independent trials.

Fig. 1 — Latency over time. Fixed ZF latency rises unbounded as channel conditions degrade. WaveLynk switches to MRT at t ≈ 22 s, immediately stabilizing latency to ~44 ms. Error bars: n = 5 repetitions.
Fig. 2 — Latency and packet loss vs. received signal power. At weak signal (−80 dBm), fixed ZF collapses while WaveLynk maintains stable performance. Both strategies converge at strong signal where CCI stays below γ.
Fig. 3 — Ping latency vs. signal power. MRT-dominant mode (WaveLynk when CCI ≥ γ) consistently outperforms ZF-dominant mode at signal levels below −65 dBm — the coherence cliff transition zone.
Fig. 4 — Packet loss rate vs. signal power. At −80 dBm, ZF-dominant mode shows 45% packet loss vs. 18% for MRT-dominant. WaveLynk's adaptive strategy captures the MRT benefit exactly when needed.
Methodology

Hardware testbed setup

Equipment

  • Router: Netgear Nighthawk Wi-Fi 6/7, 6 GHz band
  • Server: 1 laptop running iperf3 as throughput server
  • Clients: 2 laptops running iperf3, stressing MU-MIMO
  • Monitors: 2 smartphones measuring live RSSI and throughput
  • Trials: 100 independent measurement runs

Experimental controls

  • Same client device and router for all trials
  • Fixed distance and antenna orientation per trial
  • Same channel and bandwidth throughout
  • Identical traffic pattern (iperf3 UDP stream)
  • Each measurement repeated n = 5 times; error bars shown
  • Signal strength varied by physical positioning
Comparison protocol: each signal strength condition was tested under fixed ZF mode (ZF at all times), fixed MRT mode (MRT at all times), and WaveLynk adaptive mode (CCI-based switching with γ = 0.6). Strategies used identical hardware and channel conditions — only the precoding mode differed.

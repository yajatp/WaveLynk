# WaveLynk Data Directory

This directory contains both simulation outputs and hardware measurement data
used in the WaveLynk research paper.

---

## Directory Structure

```
data/
├── hardware/           ← Real-world measurement data from testbed
│   ├── packet_loss.csv
│   ├── latency_ms.csv
│   └── throughput_mbps.csv
└── simulation/
    └── monte_carlo_results.csv  ← Generated by notebooks/03_monte_carlo.ipynb
```

---

## Hardware Data (`hardware/`)

### Collection Method

Data was collected using the following Wi-Fi 6/7 testbed:

- **Router:** Netgear Nighthawk Wi-Fi 6/7 router operating on the 6 GHz band
- **Server:** 1 laptop running `iperf3` as the server
- **Clients:** 2 laptops running `iperf3` as clients, stressing MU-MIMO link
- **Monitors:** 2 smartphones measuring live throughput and signal quality
- **Trials:** 100 independent measurement runs
- **Comparison:** WaveLynk adaptive switching vs. always-ZF vs. always-MRT

### CSV Column Descriptions

#### `packet_loss.csv`

| Column | Type | Description |
|--------|------|-------------|
| `trial` | int | Trial number (1–100) |
| `signal_strength_dbm` | float | Received signal strength in dBm |
| `mode` | str | Beamforming strategy: `"ZF"`, `"MRT"`, or `"WaveLynk"` |
| `packet_loss_pct` | float | Measured packet loss percentage |

#### `latency_ms.csv`

| Column | Type | Description |
|--------|------|-------------|
| `trial` | int | Trial number (1–100) |
| `signal_strength_dbm` | float | Received signal strength in dBm |
| `mode` | str | Beamforming strategy: `"ZF"`, `"MRT"`, or `"WaveLynk"` |
| `latency_p50_ms` | float | Median (P50) round-trip latency in ms |
| `latency_p95_ms` | float | 95th percentile latency in ms |
| `latency_p99_ms` | float | 99th percentile latency in ms |

#### `throughput_mbps.csv`

| Column | Type | Description |
|--------|------|-------------|
| `trial` | int | Trial number (1–100) |
| `signal_strength_dbm` | float | Received signal strength in dBm |
| `mode` | str | Beamforming strategy: `"ZF"`, `"MRT"`, or `"WaveLynk"` |
| `throughput_mbps` | float | Measured throughput in Mbps |

## Simulation Data (`simulation/`)

### `monte_carlo_results.csv`

Generated by `notebooks/03_monte_carlo.ipynb`. Contains 100-trial Monte Carlo
simulation results comparing the three beamforming strategies under random
channel conditions.

| Column | Type | Description |
|--------|------|-------------|
| `trial` | int | Trial number (1–100) |
| `velocity_mps` | float | Randomized user velocity in m/s |
| `snr_db` | float | Randomized SNR in dB |
| `strategy` | str | `"ZF"`, `"MRT"`, or `"WaveLynk"` |
| `mean_sinr_db` | float | Average SINR across simulation duration |
| `outage_rate` | float | Fraction of timesteps where SINR < threshold |
| `peak_error` | float | Maximum precoding perturbation error |
| `num_switches` | int | Number of mode switches (WaveLynk only) |

## Notes

- Hardware CSV files are added manually by the research team after data
  collection. If they are not present, see
  `notebooks/04_hardware_validation.ipynb` for the expected format and a
  demonstration using synthetic placeholder data.
- Simulation CSV files are generated programmatically by the notebooks.
  Re-run `notebooks/03_monte_carlo.ipynb` to regenerate.
